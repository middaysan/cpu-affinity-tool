# Security Policy

## Reporting a security issue

Please do not open public GitHub issues for security-sensitive problems.

Report security issues privately to:

- `midda7@gmail.com`

Include:

- CPU Affinity Tool version
- Windows version
- CPU model if relevant
- a short impact summary
- reproduction steps or proof of concept
- whether the issue requires local access, user interaction, or administrator privileges

## What counts as a security issue

Examples include:

- unintended code execution paths
- privilege escalation beyond the documented administrator requirement
- unsafe handling of shortcuts, URIs, or file paths
- release artifact tampering or packaging integrity issues
- data exposure that goes beyond normal local application behavior

## What does not belong here

Please use the normal issue templates for:

- feature requests
- UI feedback
- compatibility bugs
- launch or monitoring bugs that do not have a security impact
